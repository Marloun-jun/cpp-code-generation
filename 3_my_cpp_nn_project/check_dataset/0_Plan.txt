+ 1. Структурная целостность (Автоматизировано)
	Проверить, что все строки имеют ровно 5 полей (описание, код, стиль, тема, ключевые слова)
	Убедиться в корректности экранирования кавычек в коде (проблема с "Hello World" внутри "...")
	Проверить разделители — запятые внутри полей должны быть в кавычках
+ 2. Балансировка стилей (Быстрый анализ)
	Посчитать количество примеров для каждого style:
		using_namespace_std — должно быть ~5000
		explicit_std — должно быть ~5000
	Если дисбаланс >10%, пересмотреть распределение
+ 3. Категоризация тем (Визуальная проверка)
	Вывести список уникальных topic значений (должно быть 10-30 тем)
	Проверить отсутствие опечаток: basic_io, OOP, templates и т.д.
	Убедиться, что каждая тема представлена в обоих стилях
+ 4. Проверка описаний (Выборочно)
	Просмотреть 100-200 случайных description
	Описания должны:
		Четко соответствовать коду
		Быть уникальными (не повторяться дословно)
		Начинаться с глагола ("напиши", "создай", "реализуй")
+ 5. Ключевые слова (Автоматизированная выборочка)
	Проверить, что keywords содержат релевантные термины
	Убедиться в консистентности: для explicit_std ключевые слова должны начинаться с std::
		Пример: "std::cout,std::endl" а не "cout,endl"
+ 6. Проверка кода (Критически важный этап)
	Не компилировать всё! Вместо этого:
		Для каждой уникальной пары (topic, style) выбрать 3-5 случайных примеров
		Скомпилировать их с флагами: -std=c++17 -Wall -Wextra
		Проверить отсутствие ошибок компиляции и предупреждений
		Запустить скомпилированные программы, проверить вывод
- 6.1 Проверка версии С++20
	Разделение датасета на части, в зависимости от версии с++ (17, 20)
- 7. Контроль дублирования (Для вашего объема)
	Проверить хеши MD5/SHA1 только для поля code
	Удалить полностью идентичные примеры кода (но сохранить их пары в другом стиле!)
	Разрешить семантически похожий код (разные description, но похожий code)
+ 8. Финальная выборочная проверка
	Взять 50 случайных строк из итогового датасета
	Проверить визуально всю цепочку: описание → код → стиль → тема → ключевые слова
	Убедиться в логической связи всех компонентов
