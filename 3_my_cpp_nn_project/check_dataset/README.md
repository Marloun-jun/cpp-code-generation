# Проверка качества датасета C++ кода

Набор инструментов для валидации и анализа датасета примеров C++ кода (9300 примеров), созданного для обучения нейронной сети генерации кода.

## 📋 О датасете

**Файл:** `2_cpp_code_generation_dataset.csv`  
**Размер:** 9300 примеров  
**Структура:**

| Поле | Описание | Пример |
|------|----------|--------|
| `description` | Описание программы на русском | "Напиши программу для вычисления факториала" |
| `code` | Исходный код C++ | `#include <iostream>\nint main() { ... }` |
| `style` | Стиль использования std | `using_namespace_std` / `explicit_std` |
| `topic` | Тематическая категория | "loops" |
| `keywords` | Ключевые слова через запятую | "loops,factorial,endl,return" |

## 🔍 Программы проверки

| Программа | Назначение | Ключевые проверки |
|-----------|------------|-------------------|
| `p_1_structure.py` | **Структурная целостность** | • 5 полей в каждой строке<br>• Корректные кавычки и экранирование<br>• Допустимые значения style<br>• Непустые topic и keywords |
| `p_2_std.py` | **Балансировка стилей** | • Распределение using/explicit (цель: 50/50)<br>• Поиск нарушений: `using_namespace_std` с `std::`<br>• Автоматическое исправление (опционально)<br>• 70+ замен для std-элементов |
| `p_3_topic.py` | **Категоризация тем** | • Извлечение уникальных тем<br>• Проверка опечаток и дубликатов<br>• Баланс стилей для каждой темы<br>• Выявление проблемных тем |
| `p_4_descriptions.py` | **Качество описаний** | • Начинаются с глагола (30+ вариантов)<br>• Длина 10-500 символов<br>• Отсутствие кода в описании<br>• Выборочная проверка (200 примеров) |
| `p_5_keywords.py` | **Ключевые слова** | • Количество (2-15 оптимально)<br>• Правильные префиксы `std::`<br>• Консистентность между стилями<br>• Автоматическое исправление в `explicit_std`<br>• ~100 std-ключевых слов |
| `p_6_compilation.py` | **Компиляция кода** | • Реальная компиляция (g++/clang++)<br>• Многопоточное тестирование<br>• Классификация ошибок<br>• Детальные JSON-отчёты |

## 🚀 Быстрый старт

### Запуск всех проверок
```bash
# Поочерёдный запуск всех программ
python p_1_structure.py
python p_2_std.py
python p_3_topic.py
python p_4_descriptions.py
python p_5_keywords.py
python p_6_compilation.py
```

### Проверка и исправление ключевых слов
```bash
# Только проверка (10000 примеров)
python p_5_keywords.py

# Проверка + автоматическое исправление
python p_5_keywords.py --fix

# Исправление с сохранением в reports/
python p_5_keywords.py --fix --output reports/fixed_dataset.csv
```

### Проверка компиляции (интерактивный режим)
```bash
python p_6_compilation.py

# В меню можно выбрать:
1 - Быстрый тест (50 примеров)
2 - Средний тест (500 примеров)
3 - Полный тест (все примеры)
4 - Выборочный тест по теме
5 - Тест конкретных строк
D - Включить/выключить режим отладки
```

## 📊 Результаты проверок

### Программы генерируют отчёты в папке reports/:
| Файл | Описание | Примечание |
|------|----------|------------|
| `compilation_report_*.json` | Результаты компиляции в JSON формате | Создается по результатам проверки (`p_6_compilation.py`) |
| `fixed_dataset.csv` | Исправленная версия датасета | Создается при наличии исправлений в dataset (`p_5_keywords.py --fix`) |
| `structure_report_*.txt` | Детальный отчёт о структурных ошибках | Создается при наличии ошибок  (`p_1_structure.py`) |

### Пример отчёта p_5_keywords.py:
```text
- explicit_std без std:: у std-ключевых: 11
  └─ из них несоответствие коду (в коде с std::): 4 ← ИСПРАВЛЯЕМЫЕ
- using_namespace_std с std:: 3
  └─ из них несоответствие коду (в коде без std::): 1
```

### Итоговая оценка качества компиляции:
```text
ВЫДАЮЩЕЕСЯ КАЧЕСТВО!   (99.5%+ успешной компиляции)
ОТЛИЧНОЕ КАЧЕСТВО!      (98%+)
ХОРОШЕЕ КАЧЕСТВО        (95%+)
УДОВЛЕТВОРИТЕЛЬНОЕ      (90%+)
ТРЕБУЕТСЯ ДОРАБОТКА     (<90%)
```

## 🛠 Требования
- Python 3.8 или выше
- Компилятор C++ (g++ или clang++) для p_6_compilation.py
- Все необходимые модули являются встроенными в Python (дополнительных зависимостей не требуется)

## 📝 Примечания

- Программы только проверяют код (кроме p_2_std.py с опциональным исправлением)
- p_5_keywords.py --fix автоматически добавляет std:: к ключевым словам в explicit_std, если они используются в коде с префиксом
- Для компиляции требуется установленный компилятор C++ (g++/clang++)
- Отчёты сохраняются в папку reports/ с временной меткой
- Все программы поддерживают аргументы командной строки для гибкой настройки
- Датасет сбалансирован по стилям: ~50% using_namespace_std, ~50% explicit_std
