## 🎯 О проекте

**Цель:** Создание end-to-end решения для генерации C++ кода по русским инструкциям — от подготовки данных до веб-сервера.

**Ключевые особенности:**
- 🔤 **Собственный BPE токенизатор** — Python (эталон) + C++ (продакшен) с SIMD-оптимизациями
- 🤖 **Трансформерная модель** — 18.3M параметров, обучена на 9300+ примерах C++ кода
- 🎯 **LoRA дообучение** — понимание русских инструкций (20K+ примеров)
- 🌐 **Веб-сервер** — демонстрация генерации кода в реальном времени
- 📊 **Полный цикл** — данные → токенизатор → модель → сервер
- ✅ **Валидация** — сравнение C++ и Python реализаций токенизатора

## 📊 Ключевые результаты

### Токенизатор (BPE)

| Метрика | Python BPE | C++ BPE | Ускорение |
|---------|-----------|---------|-----------|
| Скорость (DataLoader) | ~5,100 экз/с | **~22,300 экз/с** | **~4.3x** |
| Время encode | 195 мкс | **16 мкс** | **~12x** |
| Память | 2.5 МБ | **0.1 МБ** | **25x меньше** |

### Модель генерации кода

| Модель | Параметры | Val Loss | Понимание инструкций |
|--------|-----------|----------|---------------------|
| Medium | 18.3M | 0.87 | ❌ |
| **Medium + LoRA** | 18.3M + 1.8M | **0.78** | ✅ |

### Качество генерации (лучший вариант генерации)

```cpp
#include <iostream>
int main() {
    int x = 10;
    int* p = &x;
    std::cout << *p << std::endl;
    return 0;
}
```
**Примечания:** 
- Модель не всегда генерирует синтаксически правильный C++ код и "понимает" русские инструкции (с LoRA).
- Часто в коде генерируется "мусор".
- Возможно, модель запомнила данный пример.

## 🚀 Быстрый старт

### 1. Установка

```bash
# Клонирование
git clone https://github.com/Marloun-jun/cpp-code-generation.git
cd 3_my_cpp_nn_project

# Виртуальное окружение
python -m venv ../.venv
source ../.venv/bin/activate

# Зависимости
pip install -r requirements.txt
```

### 2. Подготовка данных

```bash
# Очистка и анализ датасета
cd check_dataset
python p_1_structure.py
python p_6_compilation.py
```

### 3. Обучение токенизатора

```bash
# Python версия (эталон)
cd ../bpe_tokenizer_cpu/bpe_python
python trainer.py --vocab-size 10000

# Конвертация в C++ формат
cd ../bpe_cpp/tools
python convert_vocab.py --model-size 10000

# Сборка C++ версии
cd ..
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
```

### 4. Обучение модели

```bash
# Подготовка токенизированных данных
cd ../../../cpp_code_model/scripts
python prepare_train_dataset.py

# Обучение Medium
python train_medium.py
```

### 5. LoRA дообучение

```bash
# Подготовка датасета инструкций
python prepare_finetune_dataset.py

# Обучение русского токенизатора
cd description_tokenizator
python train_rus_tokenizer.py

# LoRA дообучение
cd ..
python train_finetune_lora.py
```

### 6. Запуск веб-сервера для демонстрации

```bash
python final_server.py
# Открыть http://localhost:8080
```

## 🐳 Docker

Контейнер запускает **два сервера**:
- **Модель генерации кода** — порт `8080` (веб-интерфейс)
- **Токенизатор (демо)** — порт `8081` с Swagger UI

### Быстрый запуск (готовый образ)

```bash
# Скачать образ
docker pull ghcr.io/marloun-jun/cpp-code-gen:latest

# Запустить контейнер (порты 8080 и 8081)
docker run -d \
    --name cpp-code-gen \
    -p 8080:8080 \
    -p 8081:8081 \
    --restart unless-stopped \
    ghcr.io/marloun-jun/cpp-code-gen:latest

# Открыть в браузере:
# - Модель:        http://localhost:8080
# - Токенизатор:   http://localhost:8081
# - Swagger UI:    http://localhost:8081/swagger
```

### Сборка из исходников

```bash
git clone https://github.com/marloun-jun/cpp-code-generation.git
cd cpp-code-generation/3_my_cpp_nn_project
docker build -t cpp-code-gen .
docker run -d --name cpp-code-gen -p 8080:8080 -p 8081:8081 --restart unless-stopped cpp-code-gen
```

### Управление контейнером

```bash
# Логи
docker logs -f cpp-code-gen

# Остановить
docker stop cpp-code-gen

# Запустить
docker start cpp-code-gen

# Удалить
docker rm -f cpp-code-gen

# Проверить статус
docker ps
```

## ⭐ Заключение

Проект демонстрирует полный цикл создания системы генерации кода — от подготовки и валидации данных до обучения модели и веб-сервера. C++ токенизатор обеспечивает производительность **~22,000 примеров/сек**, а модель Medium + LoRA понимает русские инструкции и генерирует синтаксически правильный C++ код.
