## 📊 Сравнение производительности

### **`benchmark_all.py`** — сравнение трёх реализаций

**Назначение:** Комплексное сравнение производительности трёх реализаций BPE токенизатора:
- 🤗 **HuggingFace** — библиотека tokenizers (эталон)
- 🐍 **Python BPE** — собственная реализация на Python
- ⚡ **C++ BPE** — оптимизированная реализация на C++

**Измеряемые метрики:**
- Скорость encode (токенов/сек)
- Время encode (мс)
- Время decode (мс)
- Среднее количество токенов на текст
- Размер словаря
- Использование памяти (MB)
- Частота OOV (Out Of Vocabulary)

**Результаты:** Сохраняются в `benchmark_results.json`

**Запуск:**
```bash
python benchmark_all.py
```

**Пример вывода:**
```bash
================================================================================
                                РЕЗУЛЬТАТЫ СРАВНЕНИЯ                                 
================================================================================
Метрика                      🤗 HuggingFace       🐍 Python           ⚡ C++
--------------------------------------------------------------------------------
Скорость encode (ток/сек)           45,000             15,000             64,200
Время encode (мс)                       2.30               6.80               0.16
Время decode (мс)                       2.10               5.90               0.12
Токенов на текст                        45.5               45.5               45.5
Размер словаря                         8,000              8,000                178
Память (MB)                              120                 85                 45
OOV частота (%)                         1.0%               2.0%               0.0%
================================================================================
⚡ Ускорение C++ относительно Python: 42.5x
⚡ Ускорение C++ относительно HuggingFace: 14.4x
```

## 🔧 Обучение токенизаторов

### **`train_huggingface.py`** — обучение HuggingFace токенизатора

**Назначение:** Обучение BPE токенизатора из библиотеки HuggingFace Tokenizers на корпусе C++ кода.

**Особенности:**
- Byte-level токенизация (аналог GPT-4)
- Специальные токены: <PAD>, <UNK>, <BOS>, <EOS>
- астраиваемый размер словаря

**Параметры:**
| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--vocab-size` | Размер словаря | 8000 |
| `--data-path` | Путь к файлу с данными | `../data/corpus/train_code.txt` |
| `--output` | Путь для сохранения токенизатора | `hf_tokenizer.json` |
| `--min-frequency` | Минимальная частота токена | 2 |

**Запуск:**
```bash
python train_huggingface.py
python train_huggingface.py --vocab-size 16000
python train_huggingface.py --data-path ../data/corpus/train_code.txt
```

## 🔍 Анализ и валидация

### **`inspect_vocab.py`** — анализ словаря

**Назначение:** Детальный анализ структуры файла словаря vocab.json для определения его формата и проверки корректности.

**Определяемые форматы:**
- {token: id} — Python формат обучения
- {id: token} — Python формат сохранения
- {"tokens": [...]} — C++ формат

**Анализируемые метрики:**
- Типы ключей и значений
- Диапазон и непрерывность ID
- Наличие пропусков в нумерации
- Примеры токенов
- Наличие специальных символов

**Запуск:**
```bash
python inspect_vocab.py    # автоматический поиск
python inspect_vocab.py ../bpe_cpp/models/bpe_8000/cpp_vocab.json
python inspect_vocab.py ../bpe_python/models/bpe_8000/vocab.json
```

**Пример вывода:**
```text
🔍 АНАЛИЗ ФАЙЛА: vocab.json
============================================================
🏗️ СТРУКТУРА ДАННЫХ:
   Тип: dict
   Количество записей: 8000
   Типы ключей: str
   Типы значений: int

📋 ФОРМАТ СЛОВАРЯ:
   ✅ Формат: {token: ID} (Python формат обучения)
   📌 Пример: 'int' -> 42

📊 СТАТИСТИКА ID:
   • Min ID: 0
   • Max ID: 7999
   • Уникальных ID: 8000
```

### **`validate_cpp_tokenizer.py`** — валидация C++ токенизатора

**Назначение:** Проверка полной совместимости C++ реализации с Python эталоном. Критический тест перед использованием C++ версии в продакшене.

**Проверяемые аспекты:**
- Точность кодирования (совпадение последовательностей токенов)
- Точность декодирования (восстановление исходного текста)
- Скорость работы (сравнение производительности)
- Совместимость моделей (использование конвертированных словарей)

**Требования:**
- Python модель: ../bpe_python/models/bpe_8000/vocab.json
- C++ модель: ../bpe_cpp/models/bpe_8000/cpp_vocab.json
- C++ бинарник: ../bpe_cpp/build/examples/compare_with_python

**Запуск:**
```bash
python validate_cpp_tokenizer.py
```

**Пример вывода:**
```text

============================================================
📈 Статистика:
```

## 🚀 Типичные сценарии использования

### Полный цикл сравнения
```bash
# 1. Обучить HuggingFace токенизатор (если ещё не сделано)
python train_huggingface.py --vocab-size 8000

# 2. Запустить сравнение всех трёх реализаций
python benchmark_all.py

# 3. Визуализировать результаты
python plot_results.py

# 4. Посмотреть графики в reports/figures/comparison.png
```

### Быстрая проверка Python
```bash
# Проверить производительность Python реализации
python compare_speed_fixed.py
```

### Валидация C++ реализации
```bash
# Убедиться, что C++ токенизатор работает идентично Python
python validate_cpp_tokenizer.py
```

**Анализ словаря**
```bash

python inspect_vocab.py ../bpe_cpp/models/cpp_vocab.json
```

**Интеграция с PyTorch**
```bash

```
