## 🎯 Обзор тестов

### **`test_vocabulary.cpp`** — управление словарём

Тестирует класс `Vocabulary`, обеспечивающий двустороннее отображение между токенами и их ID.

**Проверяет:**
- ✅ Добавление и получение токенов
- ✅ Обработку дубликатов
- ✅ Специальные токены
- ✅ Поиск несуществующих токенов (INVALID_TOKEN)
- ✅ Сериализацию в JSON и обратно
- ✅ Сохранение/загрузку в файлы (текстовый и бинарный форматы)
- ✅ Производительность при большом количестве токенов

---

### **`test_tokenizer.cpp`** — базовая версия BPE

Тестирует класс `BPETokenizer` — классическую реализацию BPE алгоритма.

**Проверяет:**
- ✅ Загрузку словаря из разных форматов (JSON, бинарный)
- ✅ Кодирование и декодирование различных типов текстов
- ✅ Обработку UTF-8 и byte-level режим
- ✅ Работу правил слияния (merges)
- ✅ Пакетную обработку
- ✅ Граничные случаи (пустой вход, очень длинный текст)
- ✅ Производительность encode

---

### **`test_fast_tokenizer.cpp`** — оптимизированная версия

Тестирует класс `FastBPETokenizer` — высокопроизводительную реализацию с SIMD и кэшированием.

**Проверяет:**
- ✅ Создание с разными конфигурациями
- ✅ Загрузку модели из файлов
- ✅ Кодирование различных типов данных (числа, операторы, ключевые слова)
- ✅ Кодирование UTF-8 символов (русские, китайские, эмодзи)
- ✅ Roundtrip тесты (encode + decode)
- ✅ Пакетную обработку и сравнение с последовательной
- ✅ Эффективность кэширования
- ✅ Производительность encode

---

### **`test_parallel.cpp`** — параллельное обучение

Тестирует класс `ParallelTrainer` — многопоточное обучение токенизатора.

**Проверяет:**
- ✅ Создание тренера с разным количеством потоков
- ✅ Разбиение корпуса на чанки (равномерное распределение)
- ✅ Параллельный подсчет частот символов
- ✅ Корректность подсчета на пустом и большом корпусе
- ✅ Отслеживание прогресса и отмену обучения
- ✅ Многопоточный доступ (без гонок данных)
- ✅ Производительность и масштабирование

---

### **`test_memory_pool.cpp`** — пул памяти

Тестирует класс `MemoryPool` — оптимизированный аллокатор для частых операций.

**Проверяет:**
- ✅ Базовое выделение и освобождение памяти
- ✅ Повторное использование освобожденных блоков
- ✅ Выделение памяти больше размера блока (куча)
- ✅ Множественные выделения в разном порядке
- ✅ Граничные случаи (нулевой размер, nullptr)
- ✅ Интеграцию с STL через PoolAllocator
- ✅ Сравнение аллокаторов

---

### **`test_utils.cpp`** — вспомогательные утилиты

Тестирует функции из `utils.hpp` — работа с файлами, форматирование, UTF-8.

**Проверяет:**
- ✅ Чтение и запись файлов (текстовых и бинарных)
- ✅ Обработку ошибок (несуществующий файл, защищенная директория)
- ✅ Форматирование размеров (B, KB, MB, GB, TB)
- ✅ Валидацию UTF-8 (корректные и некорректные последовательности)
- ✅ Экранирование спецсимволов
- ✅ Точность и монотонность таймера

---

### **`test_compatibility.cpp`** — совместимость с Python

Сравнивает результаты C++ токенизатора с эталонной Python реализацией.

**Проверяет:**
- ✅ Загрузку той же модели, что и Python версия
- ✅ Кодирование всех тестовых строк
- ✅ Roundtrip тесты (encode + decode)
- ✅ Пакетную обработку
- ✅ Работу с русскими комментариями и Unicode
- ✅ Прямое сравнение с сохраненными выходами Python

---

### **`test_trained_model.cpp`** — тестирование обученной модели

Загружает предварительно обученную модель и проверяет её работу.

**Проверяет:**
- ✅ Поиск и загрузку модели (бинарный или текстовый формат)
- ✅ Кодирование различных конструкций C++
- ✅ Статистику токенов (количество, уникальность)
- ✅ Производительность на большом тексте
- ✅ Многопоточное тестирование (если доступен OpenMP)
- ✅ Эффективность кэша

## 🔧 Сборка и запуск тестов

### **Сборка тестов**

```bash
# Из корневой директории проекта
cd bpe_cpp
mkdir build && cd build
cmake .. -DBUILD_TESTING=ON
make
```
### **Запуск всех тестов**
```bash
# Из директории build
ctest

# Или через make
make test

# Или с помощью скрипта
../scripts/run_tests.sh
```

### **Запуск конкретного теста**
```bash
# Запустить конкретный тестовый файл
./tests/test_vocabulary
./tests/test_tokenizer
./tests/test_fast_tokenizer
./tests/test_parallel
./tests/test_memory_pool
./tests/test_utils
./tests/test_compatibility
./tests/test_trained_model
```

### **Запуск с фильтром**
```bash
# Запустить только тесты, содержащие "Encode" в названии
ctest -R Encode

# Запустить только тесты из файла test_tokenizer
ctest -R Tokenizer

# Исключить тесты с "Performance"
ctest -E Performance
```

### **Подробный вывод**
```bash
# Подробный вывод всех тестов
ctest -V

# Еще более подробный вывод (с выводом из тестов)
ctest -VV

# Запуск с отображением прогресса
ctest --progress
```

### **Повторный запуск**
```bash
# Повторить тесты до первого провала (для поиска flaky тестов)
ctest --repeat until-fail:5

# Повторить неудачные тесты
ctest --rerun-failed
```

## 📊 Покрытие кода

Для анализа покрытия кода тестами используйте:
```bash
# Сборка с флагами покрытия
cmake .. -DCMAKE_BUILD_TYPE=Debug -DCMAKE_CXX_FLAGS="--coverage"

# Запуск тестов
ctest

# Генерация отчета (требуется gcov/lcov)
lcov --capture --directory . --output-file coverage.info
genhtml coverage.info --output-directory coverage_report
```

#### Генерация отчета о покрытии
```bash
# Через скрипт
../scripts/run_tests.sh --coverage --html
```

## 🔍 Что тестируется

### **`test_vocabulary.cpp`**
- Добавление и получение токенов (ID должны быть последовательными)
- Обработка дубликатов (возврат того же ID)
- Специальные токены (<PAD>, <UNK>, <BOS>, <EOS>)
- Поиск несуществующих токенов (INVALID_TOKEN)
- Сериализация в JSON и обратно
- Сохранение/загрузка в файлы
- Бинарный формат (сохранение всех 256 байт)

### **`test_tokenizer.cpp`**
- Загрузка из JSON и бинарного формата
- Кодирование простых и сложных строк
- Применение правил слияния (merges)
- Декодирование и roundtrip
- Пакетная обработка
- UTF-8 и byte-level режим
- Граничные случаи (пустой вход, очень длинный текст)

### **`test_fast_tokenizer.cpp`**
- Создание с разными конфигурациями
- Кодирование чисел, операторов, ключевых слов
- Кодирование UTF-8 (русские, китайские, эмодзи)
- Roundtrip тесты
- Пакетная обработка и сравнение с sequential
- Эффективность кэширования
- Производительность encode

### **`test_parallel.cpp`**
- Создание тренера с разным количеством потоков
- Разбиение корпуса на чанки
- Подсчет частот символов
- Отслеживание прогресса
- Отмена обучения
- Многопоточный доступ
- Производительность и масштабирование

### **`test_memory_pool.cpp`**
- Базовое выделение и освобождение
- Повторное использование блоков
- Крупные выделения (> BlockSize)
- Множественные выделения
- Граничные случаи

- Сравнение аллокаторов

### **`test_utils.cpp`**
