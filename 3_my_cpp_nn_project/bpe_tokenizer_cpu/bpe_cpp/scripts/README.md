## 🔧 Скрипт сборки

### **`build.sh`** — универсальный скрипт сборки

**Назначение:** Автоматизация процесса сборки проекта с CMake.

**Возможности:**
- Поддержка разных типов сборки (Debug, Release, RelWithDebInfo)
- Автоматическое определение количества ядер для параллельной сборки
- Опции для очистки, тестирования, бенчмарков и установки
- Проверка наличия необходимых инструментов (cmake, make, g++)
- Цветной вывод с эмодзи для лучшей читаемости

**Использование:**
```bash
./build.sh [options] [build_type]

# Типы сборки:
# Debug             - Отладочная сборка (без оптимизаций, с символами)
# Release           - Оптимизированная сборка (по умолчанию)
# RelWithDebInfo    - Оптимизированная с отладочной информацией

# Опции:
#   --clean         - Очистить директорию сборки перед сборкой
#   --test          - Запустить модульные тесты после сборки
#   --benchmark     - Запустить бенчмарки после сборки
#   --install       - Установить собранные файлы
#   --verbose       - Подробный вывод CMake и make
#   --help          - Показать справку
```
**Примеры:**
```bash
# Release сборка (по умолчанию)
./build.sh

# Debug сборка с очисткой
./build.sh Debug --clean

# Release сборка + тесты + бенчмарки
./build.sh --test --benchmark

# Установка после сборки
./build.sh Release --install
```
## 📊 Скрипты для бенчмарков

### **`run_benchmarks.sh`** — запуск всех бенчмарков

**Назначение:** Запуск бенчмарков производительности и генерация отчетов.

**Возможности:**

- Запуск различных типов бенчмарков (fast, original, comparison, parallel)
- Настройка количества итераций
- Экспорт в разные форматы (console, json, csv, plot, html)
- Сравнение с предыдущими результатами
- Генерация графиков (требуется matplotlib)
- Создание интерактивных HTML отчетов


**Использование:**
```bash
./run_benchmarks.sh [options]

# Опции:
#   --type TYPE        Тип бенчмарка:
#                      all           - все бенчмарки (по умолчанию)
#                      fast          - только быстрый токенизатор
#                      original      - только оригинальный
#                      comparison    - сравнение версий
#                      parallel      - многопоточность
#
#   --iterations N     Количество итераций для каждого бенчмарка
#   --format FORMAT    Формат вывода (console|json|csv|plot|html)
#   --output FILE      Сохранить результаты в файл
#   --compare          Сравнить с предыдущими результатами
#   --html             Сгенерировать HTML отчет
#   --verbose          Подробный вывод
#   --help             Показать справку
```
**Примеры:**
```bash
# Запуск всех бенчмарков с графиками
./run_benchmarks.sh --type all --format plot

# Сравнение производительности с предыдущим запуском
./run_benchmarks.sh --type comparison --compare

# Сохранение результатов в JSON
./run_benchmarks.sh --type fast --iterations 10 --output results.json

# Генерация HTML отчета
./run_benchmarks.sh --type all --html
```
### **`compare_performance.py`** — сравнение Python и C++

**Назначение:** Детальное сравнение производительности Python и C++ реализаций токенизатора.

**Возможности:**

- Генерация тестового C++ кода заданного размера
- Многократные замеры для статистической достоверности
- Прогрев (warmup) для стабилизации кэшей
- Расчет метрик: min, max, mean, median, stdev
- Проверка корректности (количество токенов должно совпадать)
- Сохранение в JSON и построение графиков

**Использование:**
```bash
# Базовое сравнение
./compare_performance.py --size 1000 --iterations 20

# Сравнение с моделью 10000 и построение графика
./compare_performance.py --model-size 10000 --plot --output results.json

# Детальное тестирование с большим текстом
./compare_performance.py --size 10000 --iterations 5 --verbose
```
**Примеры:**
```bash
# Базовое сравнение
./compare_performance.py --size 1000 --iterations 20

# Сравнение с моделью 10000 и построение графика
./compare_performance.py --model-size 10000 --plot --output results.json

# Детальное тестирование с большим текстом
./compare_performance.py --size 10000 --iterations 5 --verbose
```

## 🔍 Скрипт для тестирования

### **`run_tests.sh`** — запуск модульных тестов

**Назначение:** Запуск всех модульных тестов с расширенными возможностями анализа.

**Возможности:**

- Фильтрация тестов по имени
- Параллельный запуск для ускорения
- Многократный прогон для выявления flaky тестов
- Анализ покрытия кода (gcov/lcov)
- Проверка утечек памяти через Valgrind
- Генерация JUnit XML и HTML отчетов

**Использование:**

./run_tests.sh [options]
```bash
# Опции:
#   --filter FILTER    Фильтр тестов (например: *Tokenizer*)
#   --coverage         Анализ покрытия кода (gcov/lcov)
#   --parallel N       Параллельный запуск N потоков
#   --repeat N         Повторить тесты N раз
#   --verbose          Подробный вывод
#   --xml              JUnit XML отчет
#   --html             HTML отчет о тестах
#   --memcheck         Проверка утечек памяти (Valgrind)
#   --help             Показать справку
```
**Примеры:**
```bash
# Запуск всех тестов с подробным выводом
./run_tests.sh --verbose

# Запуск только тестов токенизатора
./run_tests.sh --filter *Tokenizer*

# Анализ покрытия с HTML отчетом
./run_tests.sh --coverage --html

# Проверка памяти с Valgrind
./run_tests.sh --memcheck --parallel 4

# Генерация XML отчета для CI/CD
./run_tests.sh --xml --output junit.xml
```

## 📈 Скрипт для профилирования

### **`profile.sh`** — профилирование производительности

**Назначение:** Запуск различных профилировщиков для анализа производительности.

**Возможности:**

- builtin - встроенный SimpleProfiler (статистика операций)
- perf - Linux perf (анализ на уровне процессора)
- callgrind - Valgrind Callgrind (симуляция кэша)
- gprof - GNU gprof (профилирование функций)
- flamegraph - генерация Flame Graphs (визуализация)
- all - все инструменты последовательно

**Использование:**
```bash
./profile.sh [options]

# Опции:
#   --tool TOOL        Инструмент профилирования:
#                          builtin|perf|callgrind|gprof|flamegraph|all
#   --target TARGET    Цель для профилирования:
#                          tokenizer|encode|decode|train
#   --size SIZE        Размер тестовых данных:
#                          small (10KB) | medium (100KB) | large (1MB) | huge (10MB)
#   --output DIR       Директория для сохранения отчетов
#   --open             Открыть отчет после генерации
#   --compare          Сравнить до/после оптимизации
#   --help             Показать справку
```

**Примеры:**
```bash
# Встроенный профайлер для encode
./profile.sh --tool builtin --target encode --size large

# Flame Graph визуализация
./profile.sh --tool flamegraph --target decode --open

# Запуск всех профайлеров и сравнение
./profile.sh --tool all --compare

# Профилирование обучения на больших данных
./profile.sh --tool perf --target train --size huge
```

## 📄 Сводная таблица

| Скрипт | Основное назначение | Форматы вывода | Когда использовать |
|--------|-------------------|----------------|-------------------|
| `build.sh` | Сборка проекта | Консоль | Каждый раз при изменении кода |
| `run_tests.sh` | Проверка корректности | Консоль, XML, HTML | После изменений, в CI/CD |
| `run_benchmarks.sh` | Измерение производительности | JSON, CSV, PNG, HTML | Для регрессионного тестирования |
| `profile.sh` | Поиск узких мест | Perf, Callgrind, SVG | При оптимизации |
| `compare_performance.py` | Сравнение Python vs C++ | JSON, PNG | Для отчетов и презентаций |

## 🚀 Типичные сценарии использования

### Для разработчика:
```bash
# 1. Собрать проект
./build.sh Debug

# 2. Запустить тесты
./run_tests.sh --filter *Fast* --verbose

# 3. Проверить производительность
./run_benchmarks.sh --type fast --format plot

# 4. Найти узкие места
./profile.sh --tool flamegraph --target encode
```
### Для CI/CD:
```bash
# Сборка и тесты с отчетами
./build.sh Release
./run_tests.sh --xml --output test_results.xml
./run_benchmarks.sh --type all --format json --output bench_results.json
```

### Для оптимизации:
```bash
# 1. Замерить текущую производительность
./profile.sh --tool builtin --output before

# 2. Внести изменения в код

# 3. Сравнить результаты
./profile.sh --tool builtin --output after --compare
```
