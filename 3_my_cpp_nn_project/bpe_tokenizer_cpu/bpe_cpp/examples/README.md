## 📖 Описание примеров
### 1. simple_example.cpp — Простой пример
**Цель:** Познакомиться с базовыми операциями токенизатора.
#### Демонстрирует:
 - Загрузку модели из файлов
 - Кодирование (encode) текста в токены
 - Декодирование (decode) токенов обратно в текст
 - Проверку roundtrip (код -> токены -> код)
 - Вывод статистики по токенам
#### Поддерживаемые токенизаторы:
 - BPETokenizer (базовая реализация)
 - FastTokenizer (оптимизированная версия)
#### Запуск:
```bash
# Базовый режим (один пример, BPETokenizer)
./simple_example

# С подробным выводом и FastTokenizer
./simple_example --fast --verbose

# Тестирование всех примеров
./simple_example --all
```
**Когда использовать:** Идеально для первого знакомства с библиотекой.
### 2. batch_example.cpp — Пакетная обработка
**Цель:** Показать преимущества пакетной обработки нескольких текстов одновременно.
#### Демонстрирует:
 - Сравнение одиночной и пакетной обработки
 - Статистику по каждому примеру
 - Эффективность кэширования при повторяющихся паттернах
 - Зависимость производительности от размера батча
 - Сравнение с FastTokenizer (опционально)
#### Ключевые тесты:
 - 40+ примеров C++ кода разной сложности
 - Батчи размером от 1 до 100 текстов
 - Повторяющиеся паттерны для теста кэша
#### Запуск:
```bash
./batch_example
```
**Когда использовать:** Если нужно обрабатывать много текстов одновременно (например, при обучении нейросетей).
### 3. fast_tokenizer_demo.cpp — Демо оптимизированной версии
**Цель:** Продемонстрировать все возможности FastBPETokenizer и SIMD-оптимизаций.
#### Демонстрирует:
 - Проверку поддержки SIMD (AVX2, AVX, SSE4.2)
 - Базовое кодирование/декодирование
 - Пакетную обработку
 - Эффективность кэширования
 - Тест производительности на разных размерах данных
 - Прямой вызов SIMD-функций encode
 - Детальную статистику работы
#### Особенности:
 - Автоматическое определение поддержки SIMD процессором
 - Предупреждения об опасных комбинациях (флаг есть, но процессор не поддерживает)
 - Подробная статистика кэширования
#### Запуск:
```bash
# Полная демонстрация
./fast_tokenizer_demo

# Быстрый режим (пропустить медленные тесты)
./fast_tokenizer_demo --quick

# С подробным выводом
./fast_tokenizer_demo --verbose
```
**Когда использовать:** Для изучения производительности и оптимизаций.
### 4. train_example.cpp — Обучение токенизатора
**Цель:** Показать полный цикл обучения токенизатора на корпусе текстов.
#### Демонстрирует:
 - Загрузку корпуса из файла (или создание демо-данных)
 - Параллельное обучение с использованием OpenMP
 - Прогресс-бары и таймеры
 - Валидацию обученной модели
 - Сохранение в текстовом (JSON/TXT) и бинарном форматах
 - Тест загрузки сохраненной модели
#### Ключевые возможности:
 - Автоматический поиск файлов корпуса по разным путям
 - Детальная статистика корпуса (размер, уникальные символы)
 - Валидация roundtrip на случайной выборке
 - Сравнение размеров файлов
#### Запуск:
```bash
# Обучение на реальном корпусе
./train_example --corpus ../data/corpus/train_code.txt --vocab-size 10000 --validate

# Быстрый режим для тестирования
./train_example --quick

# Сохранение в бинарном формате
./train_example --save-binary
```
**Когда использовать:** Для создания собственных моделей под конкретную задачу.

### 5. `compare_with_python.cpp` — Валидация C++ токенизатора

**Цель:** Проверка корректности C++ реализации путём сравнения с Python эталоном.

#### Демонстрирует:
- Двустороннюю совместимость C++ и Python версий
- Режим encode (текст → JSON с токенами)
- Режим decode (JSON с токенами → текст)
- Тихий режим для интеграции со скриптами

#### Запуск:
```bash
# Encode режим
./compare_with_python input.txt vocab.json merges.txt

# Decode режим
./compare_with_python tokens.json vocab.json merges.txt --decode

# Тихий режим (только результат)
./compare_with_python input.txt vocab.json merges.txt --quiet
```

**Когда использовать:** Для автоматизированного тестирования и валидации через Python скрипты.

## 🛠️ Сборка примеров

```bash
cd bpe_cpp/build
cmake .. -DBUILD_EXAMPLES=ON
make examples
```
Или просто make, если примеры включены по умолчанию.
#### Сборка отдельного примера
```bash
cd bpe_cpp/build
make simple_example
make batch_example
make fast_tokenizer_demo
make train_example
```
## 🎮 Запуск примеров
После сборки все исполняемые файлы находятся в build/examples/:
```bash
cd bpe_cpp/build

# Запуск из директории build
./examples/simple_example
./examples/batch_example
./examples/fast_tokenizer_demo
./examples/train_example

# Или копирование в корень examples/
cp examples/* ../examples/
cd ../examples
./simple_example
```

## 🔧 Модели для обучения
Для работы примеров требуются файлы обученной модели:
#### Основные пути поиска:
 - models/bpe_10000/cpp_vocab.json и models/bpe_10000/cpp_merges.txt
 - bpe_cpp/models/bpe_10000/cpp_vocab.json
 - ../../bpe_python/models/bpe_10000/vocab.json
#### Если модели нет:
1. Обучите модель с помощью train_example.cpp
2. Или сконвертируйте Python-модель:
```bash
python tools/convert_vocab.py
```
