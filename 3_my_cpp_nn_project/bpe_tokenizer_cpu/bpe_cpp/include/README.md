## 🔧 Конфигурационные файлы

| Файл | Назначение | Примечание |
|------|------------|------------|
| `config.h.in` | Шаблон для генерации config.h | **Не включать напрямую!** Обрабатывается CMake |
| `config.h` | Сгенерированный конфигурационный файл | Создаётся в директории сборки, не в репозитории |
| `config.hpp` | Пользовательские настройки | Можно редактировать вручную |

**Важно!** 
- `config.h` генерируется CMake и содержит макросы типа `USE_AVX2`, `USE_OPENMP`
- `config.hpp` содержит пользовательские константы (`DEFAULT_VOCAB_SIZE`, `CACHE_SIZE_LIMIT`)

## 📦 Основные классы токенизаторов

| Файл | Класс | Описание |
|------|-------|----------|
| `bpe_tokenizer.hpp` | `BPETokenizer` | Базовая реализация BPE алгоритма. Подходит для обучения и отладки |
| `fast_tokenizer.hpp` | `FastBPETokenizer` | **Рекомендуется для продакшена**. Оптимизированная версия с SIMD, кэшем и пулом памяти |

**Когда что использовать:**
- Для обучения новых моделей -> `BPETokenizer` (проще отлаживать)
- Для инференса в продакшене -> `FastBPETokenizer` (быстрее в 5-10 раз)

## 🚀 Оптимизации и производительность

| Файл | Компонент | Что делает |
|------|-----------|------------|
| `simd_utils.hpp` | SIMD | Векторные инструкции (AVX2, SSE4.2) для массовой обработки |
| `memory_pool.hpp` | Пул памяти | Уменьшает количество аллокаций для мелких объектов |
| `thread_safe_cache.hpp` | Кэш | LRU-кэш для результатов encode с hit rate 60-80% |
| `optimized_types.hpp` | Типы данных | `merge_key_t` - 64-битный ключ вместо пары строк |
| `parallel_trainer.hpp` | Обучение | Параллельное обучение на многоядерных процессорах |
| `profiler.hpp` | Профилировщик | Поиск узких мест в производительности |

## 📊 Утилиты и вспомогательные классы

| Файл | Компонент | Функциональность |
|------|-----------|------------------|
| `vocabulary.hpp` | `Vocabulary` | Двустороннее отображение токен ↔ ID |
| `utils.hpp` | `Timer`, файлы, UTF-8 | Таймер, чтение/запись файлов, валидация UTF-8 |
| `bpe_export.hpp` | `ModelExport` | Интерфейс для сохранения/загрузки моделей в разных форматах |

## 🛠️ Порядок включения

**Рекомендуемый порядок включения заголовочных файлов в `.cpp`:**

```cpp
// 1. Конфигурация (всегда первой)
#include "config.h"      // сгенерированный CMake
#include "config.hpp"    // пользовательские настройки

// 2. Оптимизированные типы и утилиты
#include "optimized_types.hpp"
#include "utils.hpp"

// 3. Компоненты производительности
#include "memory_pool.hpp"
#include "thread_safe_cache.hpp"
#include "simd_utils.hpp"

// 4. Основные классы
#include "vocabulary.hpp"
#include "bpe_tokenizer.hpp"     // или fast_tokenizer.hpp
#include "parallel_trainer.hpp"

// 5. Экспорт (если нужен)
#include "bpe_export.hpp"
```

## 📁 Структура каталога
```text
include/
├── bpe_export.hpp           # Интерфейс экспорта/импорта моделей
├── bpe_tokenizer.hpp        # Базовая версия BPE токенизатора
├── config.h                 # Сгенерированный CMake (в build/include/)
├── config.h.in              # Шаблон для генерации config.h (CMake)
├── config.hpp               # Пользовательские конфигурационные параметры
├── fast_tokenizer.hpp       # Оптимизированная версия токенизатора
├── memory_pool.hpp          # Пул памяти для быстрых аллокаций
├── optimized_types.hpp      # Компактные типы данных (merge_key_t и др.)
├── parallel_trainer.hpp     # Параллельное обучение токенизатора
├── profiler.hpp             # Встроенный профайлер производительности
├── README.md                # Этот файл
├── simd_utils.hpp           # SIMD-оптимизированные утилиты (AVX2, SSE4.2)
├── thread_safe_cache.hpp    # Потокобезопасные реализации кэша
├── utils.hpp                # Общие утилиты (таймер, работа с файлами, UTF-8)
└── vocabulary.hpp           # Управление словарём токенов
```
