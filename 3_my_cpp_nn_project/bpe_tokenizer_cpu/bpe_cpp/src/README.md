## 📊 Обзор компонентов

### **`vocabulary.cpp`** — управление словарём

**Назначение:** Реализация двустороннего отображения между токенами и их ID.

**Ключевые особенности:**
- `id_to_token_` - вектор для O(1) доступа по ID
- `token_to_id_` - хеш-таблица для O(1) поиска по токену
- Поддержка двух форматов сериализации:
  - JSON (текстовый, читаемый)
  - Бинарный (быстрый, компактный)

**Используется в:** `bpe_tokenizer.cpp`, `fast_tokenizer.cpp`, `parallel_trainer.cpp`

---

### **`bpe_tokenizer.cpp`** — базовая реализация BPE

**Назначение:** Классическая реализация алгоритма Byte Pair Encoding.

**Режимы работы:**
- **Обычный режим** — работа с символами UTF-8
- **Byte-level режим** — работа с сырыми байтами (256 базовых токенов)

**Основные методы:**
- `encode()` / `decode()` — токенизация и детокенизация
- `train()` — обучение на корпусе текстов
- `load_from_files()` / `save_to_files()` — загрузка/сохранение

**Сложность:** O(n) для encode, O(n log n) для обучения

---

### **`fast_tokenizer.cpp`** — оптимизированная версия

**Назначение:** Высокопроизводительная реализация с современными оптимизациями.

**Оптимизации:**
- **SIMD (AVX2)** — векторная обработка 32 символов за раз
- **Lookup table** — прямое отображение char -> ID
- **Кэширование** — StringViewCache для частых слов
- **OpenMP** — параллельная обработка батчей
- **Пул памяти** — уменьшение аллокаций

**Производительность:**
- Ускорение encode: до 5-10x по сравнению с базовой версией
- Hit rate кэша: 60-80% для типичных текстов
- Масштабирование: почти линейное до 16 ядер

---

### **`parallel_trainer.cpp`** — параллельное обучение

**Назначение:** Многопоточное обучение для ускорения обработки больших корпусов.

**Алгоритмы:**
- Разбиение корпуса на независимые чанки
- Параллельный подсчет частот символов и пар
- Параллельное применение слияний
- Прогресс-бары и сбор статистики

**Использует:** OpenMP для автоматического распараллеливания

---

### **`bpe_export.cpp`** — экспорт/импорт моделей

**Назначение:** Сохранение и загрузка моделей в различных форматах.

**Поддерживаемые форматы:**
| Формат | Назначение | Преимущества |
|--------|------------|--------------|
| JSON (единый) | Отладка | Читаемый, редактируемый |
| HuggingFace | Совместимость | Интеграция с transformers |
| SentencePiece | Альтернатива | Простой текстовый формат |

---

### **`python_bindings.cpp`** — Python биндинги

**Назначение:** Предоставление C++ токенизатора из Python через pybind11.

**Возможности:**
- Полный доступ ко всем функциям FastBPETokenizer
- Автоматическая конвертация типов (vector ↔ list)
- Статистика работы в виде Python dict
- Перенаправление C++ вывода в Python

**Пример использования:**
```python
from bpe_tokenizer_cpp import FastBPETokenizer
tokenizer = FastBPETokenizer(10000, True)
tokenizer.load("vocab.json", "merges.txt")
tokens = tokenizer.encode("int main()")
```

### **`utils.cpp`** — вспомогательные утилиты

**Назначение:** Набор общих утилит, используемых во всем проекте.

**Функции:**
- 📁 Работа с файлами — read_file(), write_file()
- 📏 Форматирование — format_size() (B, KB, MB, GB, TB)
- 🔤 UTF-8 валидация — is_valid_utf8() с полной проверкой
- 🔧 Экранирование — escape_string() для отладки

## 🔧 Сборка и линковка

Все файлы собираются в единую библиотеку libbpe_tokenizer.a:
```cmake
# Из корневого CMakeLists.txt
add_library(bpe_tokenizer STATIC
    src/vocabulary.cpp
    src/bpe_tokenizer.cpp
    src/fast_tokenizer.cpp
    src/parallel_trainer.cpp
    src/bpe_export.cpp
    src/utils.cpp
)

# Python биндинги собираются отдельно
pybind11_add_module(bpe_tokenizer_cpp src/python_bindings.cpp)
target_link_libraries(bpe_tokenizer_cpp PRIVATE bpe_tokenizer)
```
## 📊 Сравнение производительности

| Компонент | Базовая версия | Оптимизированная | Ускорение |
|-----------|---------------|------------------|-----------|
| `vocabulary.cpp` | O(1) | O(1) | - |
| `bpe_tokenizer.cpp` | ~2.6 мс / 128KB | - | базовая линия |
| `fast_tokenizer.cpp` | - | ~0.015 мс / 128KB | **~175x** |
| Пакетная обработка (batch=64) | ~2.76 мс | ~0.047 мс | **~59x** |
| Декодирование | 8.28 мкс/оп | 2.75 мкс/оп | **~3x** |
| Пиковая память | ~12.6 МБ | ~10.5 МБ | -17% |

*Данные из реальных бенчмарков на Intel i5-1240P (16 ядер, 4.4 GHz)*
